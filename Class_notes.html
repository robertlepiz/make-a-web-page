Lesson Two

Creating a Structured Document

HTML files are the structure of web pages, while CSS are what dictates what the pages look like.
Javascript are the components that change the elements of the HTML. It is important to structure a web page because it becomes much easier to read and to alter the code later. When making your own web page, it's a good idea to look at other people's code in order to better understand how to write your own code.

Tree-Like Structures

DOM means Document Object Model and is the tree-like structure of a web page and HTML turns tags into elemts in the tree. All of the elements in a web page are boxes. When designing a web page from a template, the easiest way to start is by diviing all of the items into boxes. It is important to start with the biggest boxes and then proceed to the smaller ones.

Lesson Three

Adding CSS Style and HTML Structure

Cascading Style sheets are the laws that the HTML page has to follow. These properties can be applied to both individual elemets as well as to its descendants. CSS can be called from the HTML page itself or from its own seperate file. It is ideal to write CSS code on a seperate file.

ID's, Classes and Semantic tags

ID's and classes mark elements in order to chage their characteristics in CSS. While classes refer to multiple elemets, ID's refer to unique elements. Headings like h1 and h2 also help avoid to repetition by allowing you to change multiple elemets using CSS.

The Box Model

HTML elements are boxes and each box has 4 components; content, padding, border and margin. It is difficult to fit all of these boxes together on a web page. There are two ways fo dealing with this issue. The first is to set the sizes in terms of a percentange. The second option is to set the box-sizing atribute to border-box for every element. In order to have thow boxes next to each other, it is necessary to add disply:flex to your CSS code.
